# 산출물 스키마 — `output/YYYY-MM/<회사>_시계열_YYYY-MM-DD.xlsx`

## 시트 구성

| 시트 | 내용 | 컬럼 |
|---|---|---|
| `long_data` | 분석용 long-format | 회사명 / 사업연도 / 구분 / 재무제표 / 계정 / 값 |
| `별도_재무상태표` | 별도 BS 피벗 | 계정 × 사업연도 |
| `별도_포괄손익계산서` | 별도 IS 피벗 | 계정 × 사업연도 |
| `별도_자본변동표` | 별도 SCE 피벗 | 계정 × 사업연도 |
| `별도_현금흐름표` | 별도 CF 피벗 | 계정 × 사업연도 |
| `연결_재무상태표` | 연결 BS 피벗 | 계정 × 사업연도 |
| `연결_포괄손익계산서` | 연결 IS 피벗 | 계정 × 사업연도 |
| `연결_자본변동표` | 연결 SCE 피벗 | 계정 × 사업연도 |
| `연결_현금흐름표` | 연결 CF 피벗 | 계정 × 사업연도 |
| `raw_재무상태표` | PDF별 원본 표 통합 | (사업연도, 구분, 과목, 값들…) |
| `raw_포괄손익계산서` | 〃 | 〃 |
| `raw_자본변동표` | 〃 | 〃 |
| `raw_현금흐름표` | 〃 | 〃 |

## 표준 계정명 (피벗 시트의 row)

`consolidate.py`의 `KEY_ITEMS`가 정규식으로 raw 라벨을 다음 표준명으로 정규화한다.

**재무상태표**: 유동자산 / 비유동자산 / 자산총계 / 유동부채 / 비유동부채 / 부채총계 / 자본금 / 자본총계

**포괄손익계산서**: 매출액 / 매출원가 / 매출총이익 / 판매비와관리비 / 영업이익 / 법인세차감전순이익 / 법인세비용 / 당기순이익 / 총포괄이익

**현금흐름표**: 영업활동현금흐름 / 투자활동현금흐름 / 재무활동현금흐름 / 기말현금

(자본변동표는 회사별 항목 편차가 커서 표준명 미정의 — raw만 보존)

## 단위

원본 PDF 단위 그대로 (대부분 KRW 원). 변환 안 함. 분석 시 억 단위 변환은 분석 측에서.

## 사업연도 라벨

`YYYY` (정수). **보고기간 종료일 기준**. 공시일이 아님.
- 예: 2024년 4월 9일 공시 감사보고서 → 사업연도 라벨 = `2023` (2023.12.31 결산)

## 파일명 규약

`<회사명>_시계열_YYYY-MM-DD.xlsx` — `YYYY-MM-DD`는 추출 실행 날짜. 같은 회사를 다른 날 다시 추출하면 같은 폴더에 새 파일 생성 (덮어쓰기 X).

## 회사명 정규화

- "주식회사 X" → "X"
- "주식회사 X와 그 종속기업" (연결) → "X"
- 폴더명·파일명에는 위 정규화된 이름 사용.
