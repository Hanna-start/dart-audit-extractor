# DART 감사보고서 재무 파이프라인 — 프로젝트 헌법

## 정의
DART에 공시된 감사보고서 PDF를 읽어 재무제표를 표준 산출물(시계열 Excel)로 만드는 오프라인 파이프라인. 자료수집 단계가 시작점이고, 분석·리포트 단계로 확장된다.

## I/O 경로 규약 (절대)
- `_input/raw/<회사>/*.pdf` — 원본 PDF. **읽기 전용**, 어떤 도구로도 수정·삭제·이동 금지.
- `_workspace/<회사>/` — 중간 산출물 (PDF별 Excel, 검증 로그). 자유 쓰기.
- `output/YYYY-MM/<회사>_시계열_*.xlsx` — 최종 산출물. 타임스탬프(`YYYY-MM-DD`) 붙여 저장. 덮어쓰기 금지, 새 파일로 만든다.

## 결정성 분리 (절대)
숫자 계산·합계 대사·회계등식 검증은 LLM이 직접 수행하지 않는다. `.claude/skills/dart-pipeline/assets/`의 결정적 스크립트(`validate.py` 등)와 프로젝트 루트의 `dart_client.py`/`pdf_to_excel.py`/`consolidate.py`/`extractor.py`만 호출한다.

## 가정·매핑 변경 (절대)
회사별 계정 매핑·정규식 오버라이드는 `.claude/skills/dart-pipeline/references/assumptions/<회사>/` 에 기록한다. 코드 안에 하드코딩하지 않는다.

## 환각 통제
- 계산식·계정 매핑이 명시되어 있지 않으면 추측하지 말고 작업을 중단한 뒤 사용자에게 질의한다.
- 추출 결과에 누락(예: 매출액 None)이 있으면 임의로 채우지 않는다. 로그에 기록하고 보고한다.
- 회계등식이 깨진 채로 산출물을 확정하지 않는다.
